### 🧮 Updated Mathematical Logic
You must now integrate the 'Attack' time constant into the gain smoothing logic. Previously, we used 'Instant Attack'; now, the gain coefficient (gc) must transition to the target gain (gt) over the specified attack duration.

1. **Calculate Attack Coefficient (α_attack):**
   - α_attack = exp(-1.0 / (τ_attack * fs))
   - *Where τ_attack is user input in seconds and fs is sample rate.*

2. **Update Smoothing Loop:**
   If gt < gc (Limiter is engaging):
     gc = gc * α_attack + gt * (1 - α_attack)
   If gt >= gc (Limiter is releasing):
     gc = gc * α_release + gt * (1 - α_release)
